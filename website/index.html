Cuti

Provider Runtime & OpenClaw Deployment Console

Run native AI coding tools in repeatable local containers, with Claude Code by default and turnkey OpenClaw gateway, channel, browser, plugin, and voice-call operations

Get Started Documentation

🚀 Quick Start

Terminal 
# Install Python package from PyPI
uv tool install cuti

# Launch the default Claude Code runtime
cuti container

That's it! You now have a fully configured AI-powered coding environment.

✨ Key Features

Containerized Development

Native provider CLIs in an isolated Docker environment with persistent runtime state

Provider Runtime

Enable Claude Code, Codex, OpenClaw, Hermes, and OpenCode with provider-specific auth and config mounts

Native CLI Coordination

Use each provider's own session, background, and automation surfaces instead of a duplicated agent loop

Read-Only Ops Console

Launch with cuti web to inspect provider readiness, workspace drift, and recent activity

OpenClaw Deployment

Operate OpenClaw gateway, channels, browser, plugins, voice-call, nodes, sandbox, cron, secrets, memory, and wiki flows

Persistent Provider State

Provider auth, config, skills, and runtime installs persist across container sessions

Claude Session History

Inspect Claude Code transcripts and reopen old sessions with cuti history
